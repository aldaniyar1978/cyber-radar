STEP-BY-STEP SECURITY GUIDE (v1.0)
----------------------------------
Topic: General Vulnerability Mitigation
Target: System Administrators

1. IDENTIFICATION
   - Run the provided 'scanner_v1.py' to identify if ports 8080, 443, or 445 are exposed.
   - Command: python scanner_v1.py <your_server_ip>

2. IMMEDIATE ACTIONS
   - If port 445 (SMB) is open and not required, disable it immediately via Firewall.
   - For Linux: sudo ufw deny 445
   - For Windows: New-NetFirewallRule -DisplayName "Block SMB" -Direction Inbound -LocalPort 445 -Protocol TCP -Action Block

3. PATCHING
   - Visit the official vendor site (Microsoft/Linux/Cisco) referenced in the news article.
   - Download the latest Security Monthly Quality Rollup.

4. MONITORING
   - Enable logging for failed login attempts.
   - Set up alerts for unusual traffic patterns on identified ports.

DISCLAIMER: This guide is for educational purposes. Always test in a staging environment before applying to production.
